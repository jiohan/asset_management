## 1. 인증 화면 (Login / Signup)

### 스크린샷 관찰
- 전체 배경 `#F9F9F8`에 중앙 카드 배치 → 구조 자체는 정석
- 카드 타이틀("로그인", "회원가입") 텍스트가 스크린샷 상 거의 보이지 않음 — 실제 렌더링에서 `CardTitle`이 배경색과 혼합되어 저대비 상태 의심
- 폼 레이블(`이메일 *`, `비밀번호 *`)은 DOM 상 존재하나 화면에서 육안으로 식별이 어려울 정도로 연함

### 문제점

#### 🔴 Critical — 앱 브랜딩 완전 부재
카드 위에 앱 이름/로고가 없다. 사용자는 로그인 화면에서 "내가 어떤 서비스에 로그인하는가"를 즉시 인지해야 한다. 현재 CardTitle "로그인" 한 단어만 있는 상태는 서비스 신뢰감을 크게 떨어뜨린다.

**권장 개선:**
```
[로고 또는 앱 이름: 가계부]   ← 카드 밖 상단 또는 카드 헤더 상단
─────────────────────────────
로그인
```

#### 🔴 Critical — 페이지 `<title>` 태그 = "Create Next App"
브라우저 탭, 북마크, 스크린리더 모두 "Create Next App"을 읽는다. Next.js 기본값이 그대로 방치된 상태. 즉시 수정 필요.

**권장:**
`layout.tsx`에 `export const metadata = { title: '가계부' }` 설정.

#### 🟠 High — Google 버튼에 Google 아이콘 없음
"Google로 계속하기" 텍스트만 존재. Google 브랜드 가이드라인상 공식 아이콘을 사용해야 하고, 사용자 신뢰도에도 영향. `@svg/google-icon` 또는 SVG 인라인으로 추가 필요.

#### 🟠 High — 로그인 에러 메시지 시인성 낮음
"이메일 또는 비밀번호가 올바르지 않습니다." 텍스트가 비밀번호 필드와 로그인 버튼 사이의 좁은 공간에 끼어 있어 스크린샷에서 거의 보이지 않는다 (tiny `•` 점 처럼 보임). 에러 메시지는 명확히 구분되는 영역에 배치해야 한다.

**권장:**
```jsx
// 현재: 작은 rose-600 텍스트 한 줄
// 권장: 붉은 배경 배너 또는 아이콘+텍스트 조합
<div className="flex items-center gap-2 rounded-md bg-rose-50 border border-rose-200 px-3 py-2 text-sm text-rose-700">
  <AlertCircle size={16} />
  이메일 또는 비밀번호가 올바르지 않습니다.
</div>
```

#### 🟡 Medium — 비밀번호 show/hide 토글 없음
`type="password"` 입력에 눈 아이콘 토글이 없다. 특히 모바일에서 오타 확인이 불가하며 UX 표준 패턴에서 누락된 항목.

#### 🟡 Medium — 회원가입 폼 레이아웃
4개 필드(이메일, 닉네임, 비밀번호, 비밀번호 확인)가 모두 열거되어 있으나 섹션 구분 없이 나열됨. 닉네임과 이메일 그룹, 비밀번호 그룹으로 시각적 구분이 있으면 인지 부하 감소.

## 2. 대시보드 (Dashboard)

### 스크린샷 관찰
- 4개 통계 카드, 트렌드 바차트, 카테고리/계좌 2열 그리드, 최근 거래 섹션으로 구성
- 총자산 `₩-10,000` (음수) 표시 — 색상 구분 없음
- 차트 영역에 데이터 1건(지출)만 있어 막대 하나가 오른쪽에 고립
- 카테고리 차트 자리에 "이번 달 지출 내역이 없습니다" 텍스트만
- 페이지 좌상단에 "대시보드" h1 없음 (스크린샷 기준 헤더 공간이 비어 있음)

### 문제점

#### 🔴 Critical — 총자산 음수 표시에 색상 미적용
`₩-10,000` 값이 음수임에도 StatCard에 `valueColor` props가 미전달되어 기본 `text-gray-900`으로 표시된다. 사용자는 순간적으로 자산이 마이너스인지 인지하기 어렵다.

```typescript
// 현재 코드 — valueColor 미전달
<StatCard label="총자산" value={formatKRW(totalAssets)} />

// 권장 — 음수 시 rose-600 적용
<StatCard
  label="총자산"
  value={formatKRW(totalAssets)}
  valueColor={totalAssets < 0 ? 'text-rose-600' : 'text-gray-900'}
/>
```

#### 🟠 High — 통계 카드 레이블의 `uppercase tracking-wide` 한국어 비적합
```jsx
<p className="text-xs font-medium text-gray-400 uppercase tracking-wide mb-3">
  {label}  {/* "총자산", "이번 달 수입" */}
</p>
```
한국어에서 `text-transform: uppercase`는 무효(한글은 대소문자 없음)이고, `letter-spacing` 확대는 가독성을 해친다. 영문 전용 스타일을 한국어 텍스트에 그대로 적용한 상태.

**권장:** `uppercase tracking-wide` 제거. `text-xs font-medium text-gray-400`만 유지.

#### 🟠 High — 대시보드 헤더 영역 식별 어려움
"대시보드" h1(`text-xl font-semibold`)이 월 선택기와 나란히 배치되어 있으나, 사이드바의 "가계부" 앱 이름과 함께 보면 어느 것이 현재 페이지 제목인지 빠르게 파악하기 어렵다. 더 큰 페이지 헤더 또는 상단 바가 필요하다.

#### 🟠 High — 카테고리 차트 빈 상태가 계좌 요약 카드와 높이 불일치
`lg:grid-cols-2`로 나란히 배치할 때, 카테고리 차트에 데이터가 없으면 텍스트 한 줄만 있는 짧은 카드 vs 계좌 목록이 있는 긴 카드가 되어 레이아웃 불균형 발생. 빈 상태에도 최소 높이를 고정하거나 illustration을 추가해 균형 유지 필요.

#### 🟡 Medium — 트렌드 차트 범례가 차트 아래 분리 배치
차트와 범례("수입", "지출") 사이 거리가 멀어 시각적 연결이 약함. Recharts `Legend`는 기본적으로 하단에 위치하는데, 실제 스크린샷에서 범례가 차트 아래 상당히 떨어져 보임.

#### 🟡 Medium — 최근 거래 섹션: 거래 없을 때 CTA 없음
"이번 달 거래 내역이 없습니다"라는 텍스트만 표시. 사용자 입장에서 다음 행동(거래 추가)을 유도하는 링크나 버튼이 없음. 새 사용자 온보딩 경험에서 치명적인 공백.

**권장:**
```jsx
<div className="text-center py-8">
  <p className="text-sm text-gray-400 mb-3">이번 달 거래 내역이 없습니다.</p>
  <Link href="/transactions" className="text-sm text-indigo-600 hover:underline">
    첫 거래 기록하기 →
  </Link>
</div>
```

#### 🟡 Medium — 월 선택기에서 "다음 달" 버튼만 비활성화, 이유 불명확
현재 달 이후로 이동 불가 시 `disabled` 처리되어 있는데, 사용자는 왜 비활성인지 즉시 알기 어렵다. tooltip 또는 `aria-label="현재 달 이후로 이동 불가"` 추가 권장.

## 3. 거래 내역 (Transactions)

### 스크린샷 관찰
- 상단에 월/계좌/타입/카테고리 4개 필터 (native `<select>`)
- 거래 목록이 완전히 비어 있음 (빈 상태)
- 우상단에 작은 "거래 추가" 버튼 (h-8)
- 페이지 제목 "거래 내역"이 스크린샷에 보이지 않음

### 문제점

#### 🔴 Critical — 필터 select 요소에 레이블 없음
4개의 `<select>` 요소가 레이블 없이 나란히 배치됨. 사용자는 각 드롭다운이 무엇을 필터링하는지 "선택해보기 전까지" 알 수 없다.

스크린샷 분석: 첫 번째 드롭다운은 "2026 4"처럼 날짜가 보이나, 나머지 3개는 빈 상태. 이 상태에서 각 select의 목적이 불분명.

**권장:**
```jsx
<div className="flex items-center gap-2">
  <div className="flex flex-col gap-1">
    <label className="text-xs text-gray-500">기간</label>
    <select ...>월 선택</select>
  </div>
  <div className="flex flex-col gap-1">
    <label className="text-xs text-gray-500">계좌</label>
    <select ...>계좌 선택</select>
  </div>
  ...
</div>
```

#### 🟠 High — "거래 추가" 버튼 크기와 위치 비직관적
버튼이 `h-8 px-2.5 text-sm`으로 매우 작고, 페이지 우상단에 위치. 거래 입력이 이 앱의 핵심 액션임에도 CTA가 너무 약함.

비교:
- 현재: `h-8` 소형 버튼, 우상단 구석
- 권장: `h-10` 이상, Primary 스타일, 페이지 상단 헤더에 더 두드러지게 배치

#### 🟠 High — 빈 상태에 액션 없음
"조건에 맞는 거래가 없습니다." 텍스트만 있는 흰색 카드. 사용자가 다음에 뭘 해야 하는지 안내 없음.

**권장:**
```jsx
<div className="py-16 text-center space-y-3">
  <p className="text-sm text-gray-500">이번 달 거래 내역이 없습니다.</p>
  <button onClick={openDrawer} className="text-sm text-indigo-600 hover:underline">
    + 거래 추가하기
  </button>
</div>
```

#### 🟡 Medium — 거래 목록 날짜 표시 형식
날짜가 `transaction_date.slice(5)` → "03-15" 형식으로 표시됨. 월을 넘나드는 데이터가 있거나 전체 기간 필터를 사용할 때 연도 정보가 사라져 혼란 가능성.

**권장:** 선택된 월이 명확한 경우 `MM/DD`, 전체 기간은 `YYYY/MM/DD` 표시.

#### 🟡 Medium — 거래 수정/삭제 버튼 접근성
수정 버튼(`<Link>` + `<Pencil>` 아이콘), 삭제 버튼이 아이콘 형태인데 레이블 없음. 스크린리더 사용자와 초보 사용자에게 불명확.

**권장:** `title` 속성 또는 `aria-label` 추가 (현재 수정 링크에는 `title="수정"`이 있으나 삭제 버튼 확인 필요).

## 4. 계좌 관리 (Accounts)

### 스크린샷 관찰
- 계좌 하나만 표시: "기본 계좌 | 입출금 | ₩-10,000"
- 우상단 "+ 계좌 추가" 버튼
- 우측에 "수정", "비활성화" 두 버튼 나란히
- 페이지가 매우 빈 느낌 — 계좌 목록이 상단에만 있고 하단 공간이 비어 있음

### 문제점

#### 🔴 Critical — 총 자산/순자산 요약 없음
계좌 목록만 있고, 전체 총자산 합계가 페이지 어디에도 표시되지 않는다. 사용자가 계좌 관리 페이지에서 가장 보고 싶은 정보 중 하나.

**권장:** 페이지 상단에 총자산 요약 카드 추가:
```
┌─────────────────────────────────┐
│  총 자산        ₩12,500,000     │
│  사용 가능      ₩10,000,000     │
│  카드 미결제     ₩1,500,000     │
└─────────────────────────────────┘
```

#### 🟠 High — "비활성화"가 파괴적 액션인데 같은 스타일
"수정"(outline)과 "비활성화"(outline)가 완전히 동일한 버튼 스타일. 비활성화는 되돌리기 어려운 액션인데, 시각적 위계가 없어 실수 클릭 위험성 존재.

**권장:**
- 수정: `variant="outline"` (현재 유지)
- 비활성화: `variant="ghost"` + 텍스트 색상 `text-gray-400` 또는 `...` 더보기 메뉴로 숨김

#### 🟠 High — 계좌 타입 구분 아이콘 없음
"입출금", "저축", "투자", "카드" 등 계좌 타입이 텍스트로만 구분됨. 아이콘으로 빠른 시각적 구분 가능.

**권장 아이콘 매핑 (Lucide):**
- cash → `Banknote`
- checking → `CreditCard`
- savings → `PiggyBank`
- investment → `TrendingUp`
- card → `CreditCard` (다른 색상)

#### 🟡 Medium — 계좌 추가 다이얼로그: 초기 잔액 필드 UX
`type="number"` 입력에 `min={0}`은 있으나, 쉼표 구분 없는 원화 입력. 거래 추가 폼처럼 `inputMode="numeric"` + 천 단위 포맷팅 적용하면 더 일관성 있음.

## 5. 레이아웃 & 사이드바 (전체 공통)

### 문제점

#### 🔴 Critical — 컨텐츠 영역 상단 헤더 바 없음
현재 구조:
```
[사이드바 240px] | [메인 컨텐츠 — 페이지별 제목이 컨텐츠 안에 있음]
```

전문적인 SaaS/핀테크 앱 표준:
```
[사이드바 240px] | [상단 헤더 바: 페이지 제목 + CTA]
                | [메인 컨텐츠]
```

현재는 각 페이지 내부에서 h1을 직접 구현하는데, 일관성이 없고 액션 버튼(거래 추가, 계좌 추가)의 위치가 페이지마다 다름.

#### 🟠 High — 사이드바 활성 항목 표시 너무 약함
활성 상태: `bg-gray-100 text-gray-900 font-medium`

`bg-gray-100`은 `#F3F4F6`으로 배경 `#F9F9F8`과 차이가 극히 미미. 실제 화면에서 어떤 메뉴가 선택됐는지 즉각 파악하기 어려움.

**권장:**
```jsx
// 활성 상태에 left border indicator 추가
active ? 'bg-indigo-50 text-indigo-700 font-medium border-l-2 border-indigo-600' : 'text-gray-500'
```

#### 🟠 High — 사이드바 아이콘 크기 16px로 너무 작음
```jsx
<Icon size={16} />
```
Lucide 아이콘 16px는 정보 밀도가 높은 테이블에나 적합. 사이드바 내비게이션 아이콘은 최소 18~20px 권장.

#### 🟡 Medium — 사용자 영역 (닉네임 + 로그아웃) 시각 계층 불명확
현재:
- 닉네임: `text-xs text-gray-500`
- 로그아웃: `text-sm text-gray-500 hover:text-gray-900`

로그아웃 버튼이 텍스트 링크처럼 보여 클릭 가능 요소임을 직관적으로 파악하기 어려움. 사용자 아바타/이니셜과 함께 클릭 가능한 메뉴로 개선 권장.

#### 🟡 Medium — Skip Link (스킵 링크) 없음
키보드/스크린리더 사용자가 사이드바 내비게이션을 매번 탭해야 메인 컨텐츠에 도달. 접근성 표준상 "메인 컨텐츠로 이동" 스킵 링크 필요.

---

## 6. 거래 추가 드로어 (Drawer UX)

### 소스코드 분석 기반

#### ✅ 잘된 점
- 우측 슬라이드 드로어 (480px) — 컨텍스트 유지하면서 폼 입력 가능
- `transition-transform duration-300 ease-in-out` — 애니메이션 있음
- backdrop 클릭으로 닫기 — 직관적
- "저장 후 계속" 버튼 — 연속 입력 UX 배려

#### 🟠 High — 드로어 backdrop이 30% black opacity만 적용
```jsx
<div className="fixed inset-0 z-40 bg-black/30" onClick={closeDrawer} />
```
`bg-black/30`은 배경이 과하게 비쳐 보임. 전문적인 앱에서는 40~60% 권장 (`bg-black/50`).

#### 🟡 Medium — 거래 타입 버튼 그룹의 비활성 계좌 처리
이미 소스에서 처리하고 있으나, 계좌 선택 `<select>` 옵션에 "(비활성)"이 텍스트로 붙음 — 실제로 비활성 계좌는 수정 시에만 나타나므로 사용자가 비활성 계좌를 선택하려다 혼란스러울 수 있음.

#### 🟡 Medium — 금액 입력 UX (현재는 양호하나 개선 가능)
현재: 문자 입력 → 숫자만 필터링 → 천 단위 포맷팅 → hidden input 전달

잘 구현되어 있음. 다만 `placeholder="금액을 입력하세요"` 보다 `placeholder="0"` + 우측에 "원" 단위 텍스트 표시가 더 직관적.

## 7. 계좌 추가 다이얼로그

#### ✅ 잘된 점
- shadcn/ui `Dialog` 사용으로 접근성 기본 확보
- 취소/추가 버튼 순서 적절

#### 🟡 Medium — 계좌 유형에 설명 없음
"현금", "입출금", "저축", "투자", "카드" 옵션만 있고, 각 타입이 어떻게 다른지 설명 없음. 처음 사용하는 사람은 "입출금"과 "현금"의 차이를 모를 수 있음.

**권장:** 유형 선택 시 간단한 설명 표시 (tooltip 또는 helper text):
```
입출금 — 은행 통장 계좌 (이체·입출금 가능)
카드 — 신용/체크카드 (지출 기록, 총자산에서 차감)
```

## 8. 전체 디자인 시스템 관점 평가

### 색상

| 현재 사용 | 평가 | 권장 |
|-----------|------|------|
| `bg-[#F9F9F8]` 배경 | ✅ 좋음 — 눈이 편한 웜그레이 | 유지 |
| `text-emerald-600` 수입 | ✅ 적절 | 유지 |
| `text-rose-600` 지출/에러 | ✅ 적절 | 유지 |
| `bg-gray-900` Primary 버튼 | ⚠️ 너무 무채색 — 브랜드 컬러 없음 | `bg-indigo-600` 권장 (핀테크 신뢰감) |
| `text-gray-400` 레이블 | ⚠️ 대비 비율 확인 필요 — 약 3:1 수준 | `text-gray-500` 이상 또는 `text-gray-600` |

### 타이포그래피

| 요소 | 현재 | 평가 |
|------|------|------|
| 페이지 제목 h1 | `text-xl font-semibold` | ⚠️ 너무 작음 — `text-2xl` 이상 권장 |
| 카드 섹션 제목 h2 | `text-sm font-semibold text-gray-700` | ⚠️ 너무 작음 — `text-base font-semibold` |
| 총자산 금액 | `text-2xl font-semibold` | ✅ 적절 |
| 통계 카드 레이블 | `text-xs uppercase tracking-wide` | 🔴 한국어 비적합 — `uppercase tracking-wide` 제거 |

### 카드 스타일
```css
/* 현재 — 좋은 방향 */
rounded-2xl shadow-[0_8px_30px_rgb(0,0,0,0.04)] p-6
```
카드 스타일 자체는 design-rules.md의 방향과 일치. 하지만:
- 일부 페이지(계좌 관리)는 `rounded-lg border border-gray-200`을 사용해 **스타일 불일치** 발생
- 거래 목록 테이블도 `rounded-lg border`로 카드 스타일과 다름

**권장:** 모든 컨테이너를 `rounded-2xl shadow-[...]`로 통일하거나, 테이블용 스타일을 명시적으로 정의.

## 결론

Phase 3까지의 구현은 **기능적으로 견고**하고 design-rules.md의 방향을 기본적으로 따르고 있습니다. 색상, 카드 스타일, 금액 포맷 등 핵심 디자인 토큰은 일관성이 있습니다.

그러나 **"전문적인 장부 앱"으로서의 완성도**를 높이기 위해 해결해야 할 점들이 있습니다:

- 첫 인상(인증 페이지)에서 **앱 정체성 부재** — 브랜딩, 로고, 페이지 타이틀 수정이 가장 긴급
- 빈 상태가 전반적으로 **죽어 있음** — 신규 사용자 온보딩에서 다음 행동을 전혀 유도하지 않음
- 정보 계층이 약함 — **무엇이 가장 중요한 숫자인지** 화면에서 즉각 파악하기 어려움
- 접근성 기본 항목(페이지 타이틀, 필터 레이블, 스킵 링크) 누락

Phase 4(투자) 구현 전에 위 Critical/High 항목들을 정리하면, 완성도 있는 서비스로서의 기반이 더 단단해집니다.
