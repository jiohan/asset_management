# 디자인 규칙 (1페이지 규칙집)

정식 디자인 시스템이 아니다. 개발 중 일관성을 유지하기 위한 최소 기준이다.
실제 값은 개발 시작 전 Tailwind + shadcn/ui 설정과 함께 확정한다.

---

## 시각적 방향성 (Visual Direction)

> 이 섹션은 확정된 규칙이 아니라 **방향성 참고**다. 개발하면서 조정 가능.

### 핵심 방향: 미니멀 소프트 레이어링

"경계선이 아닌 깊이감(Depth)으로 정보를 구분한다."

- **Borderless Design**: border를 쓰지 않거나 배경색과 거의 차이 없는 연한 톤만 사용
- **Soft Surface**: 배경과 카드 사이 명도 차이를 최소화 → 화면 전체가 하나의 유연한 판처럼
- **Negative Space First**: 여백은 남는 공간이 아니라 정보를 강조하는 핵심 디자인 요소. 패딩은 기본보다 넉넉하게
- **숫자가 주인공**: 금액·수치는 크게, 레이블·날짜는 작고 연하게

### 배경 / 카드 톤

| 레이어 | Tailwind 기준 (참고) | 비고 |
|--------|---------------------|------|
| 페이지 배경 | `bg-[#F9F9F8]` | 순수 흰색보다 눈이 편한 웜그레이 |
| 카드 배경 | `bg-white` 또는 `bg-white/80` | 배경과 미세한 차이로 부유감 |
| 카드 그림자 | `shadow-[0_8px_30px_rgb(0,0,0,0.04)]` | 넓고 연하게 퍼지는 그림자 |
| 모서리 | `rounded-2xl` (16~24px) | 직각 배제, 부드러운 인상 |

### 뮤트 팔레트 방향

기존 규칙의 색상 4개 원칙은 유지하되, 톤을 아래 방향으로 조정 검토:

| 역할 | 기존 참고값 | 방향성 |
|------|------------|--------|
| Success (수입) | emerald-600 | 채도 낮춘 Sage Green 계열 |
| Danger (지출) | rose-600 | 바랜 Muted Rose / Terracotta 계열 |
| Primary | indigo-600 | 채도 낮춘 Steel Blue 또는 Deep Indigo |

## 색상 4개

| 역할 | 이름 | 용도 |
|------|------|------|
| Primary | 메인 브랜드 색 | 주요 버튼, 활성 탭, 링크 |
| Neutral | 그레이 계열 | 텍스트, 배경, 테두리 |
| Success | 초록 계열 | 수입 금액, 양수 표시 (+) |
| Danger | 빨간 계열 | 지출 금액, 오류 메시지, 삭제 버튼 |

> Tailwind 기본 팔레트 기준으로 확정 예정.
> 예: Primary → indigo-600, Success → emerald-600, Danger → rose-600

## 타이포그래피 (4단계)

| 단계 | 크기 | 용도 |
|------|------|------|
| Display | 24px (text-2xl) | 대시보드 주요 수치, 총자산 금액 |
| Body | 16px (text-base) | 기본 본문, 거래 목록 항목 |
| Small | 14px (text-sm) | 레이블, 보조 텍스트, 필터 옵션 |
| Tiny | 12px (text-xs) | 날짜, 메타정보, 카테고리 뱃지 |

> **한국어 주의:** 한국어 텍스트에 `uppercase`, `tracking-wide` 사용 금지. 한글은 대소문자 구분이 없어 `uppercase`가 무효이고, `letter-spacing` 확대는 가독성을 해친다.

## Spacing Scale

4px 기반 배수 사용. Tailwind 기본 spacing scale 활용.

| 용도 | 값 |
|------|-----|
| 컴포넌트 내부 패딩 | 12px (p-3) 또는 16px (p-4) |
| 섹션 간 간격 | 24px (gap-6) |
| 카드 간 간격 | 16px (gap-4) |
| 폼 필드 간 간격 | 16px (space-y-4) |

## 컴포넌트 규칙

### 버튼

| 타입 | 용도 |
|------|------|
| Primary (채워진) | 저장, 생성, 확인 등 주요 액션 |
| Secondary (테두리) | 취소, 뒤로가기 등 보조 액션 |
| Destructive (빨간) | 삭제, 계정 탈퇴 등 위험 액션 |

- 삭제 버튼은 반드시 확인 모달과 함께 사용
- **Primary 버튼 색상은 `bg-indigo-600`**. 무채색(`bg-gray-900`) 버튼을 Primary 용도로 사용하지 않는다.

### 입력창 (Input)

- 레이블은 항상 입력창 위에 표시
- 오류 메시지는 입력창 아래 빨간색으로 표시
- 필수 항목은 레이블 옆에 `*` 표시

### 카드 (Card)

- 배경: 흰색 (dark mode는 v2)
- 테두리: 사용 금지. `border border-gray-200` 방식 카드 사용 안 함
- 모서리: `rounded-2xl` 고정
- 그림자: `shadow-[0_8px_30px_rgb(0,0,0,0.04)]` 고정
- 모든 컨테이너 카드(리스트/테이블 포함)는 위 모서리·그림자로 통일

### 아이콘 크기

| 맥락 | 크기 |
|------|------|
| 사이드바 내비게이션 | 20px 이상 |
| 인라인 텍스트 옆 | 16px |
| 정보 밀도 높은 테이블 내 | 14~16px |

## 상태 색상

| 상태 | 색상 | 적용 대상 |
|------|------|-----------|
| 수입 | Success (초록) | 거래 금액, + 표시 |
| 지출 | Danger (빨강) | 거래 금액, - 표시 |
| 이체 | Neutral (회색/파랑) | 거래 금액, 이체 아이콘 |
| 오류 | Danger (빨강) | 에러 메시지, 유효성 실패 |
| 성공 | Success (초록) | 저장 완료, 인증 완료 |
| 비활성 | Neutral 흐리게 | 비활성화된 계좌/카테고리 |

## 금액 표시 규칙

- KRW 포맷: `₩1,234,567` (천 단위 콤마, 원화 기호)
- 수입: 초록색 + `+₩1,234,567`
- 지출: 빨간색 + `-₩1,234,567`
- 이체: 중립색 + `₩1,234,567` (부호 없음)
- 카드 결제 예정액: "결제 예정 ₩1,234,567" 레이블과 함께 표시
- **계산 결과 음수(총자산, 잔액 등)**: `text-rose-600` 조건부 적용. 양수/0은 기본 `text-gray-900`.

## 빈 상태 (Empty State)

- 빈 상태에는 반드시 **CTA(다음 행동 유도 버튼/링크)**를 포함한다. 텍스트만 있는 빈 상태 금지.
- 패턴:
  ```jsx
  <div className="text-center py-8">
    <p className="text-sm text-gray-400 mb-3">안내 메시지</p>
    <Link href="..." className="text-sm text-indigo-600 hover:underline">
      액션 →
    </Link>
  </div>
  ```
